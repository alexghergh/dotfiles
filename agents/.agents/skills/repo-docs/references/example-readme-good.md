# README Good Example

## Traits

- states what the project does in the first paragraph
- gives one clear way to run or try it
- shows the top-level layout only when that helps orientation
- links to deeper docs instead of duplicating them

## Example skeleton

```md
# project-name

One paragraph describing what the project does and who it is for.

## quick start
1. install dependencies
2. run the app
3. run the tests

## repository layout
- `src/` application code
- `docs/` deeper technical docs
- `scripts/` operational helpers

## further reading
- `ARCHITECTURE.md`
- `CONTRIBUTING.md`
```
