# README Bad Example

## Warning signs

- no clear statement of what the project is
- setup steps buried deep in prose
- huge architecture sections copied into the README
- low-level function detail that belongs in code docs
- stale or speculative sections that do not match the repo

## Bad pattern

```md
# project-name

This repo has existed for years and has changed a lot.

## internals
Every package, helper, and utility is described in detail here.

## maybe future plans
We might redesign the system later.
```
