# AGENTS Good Example

## Traits

- contains repo-specific workflow rules
- lists important commands that contributors or agents actually need
- states testing expectations and editing guardrails
- points to other docs instead of duplicating them

## Example skeleton

```md
# agents

## commands
- `make test`
- `make lint`

## workflow
- inspect affected files before editing
- keep patches focused
- ask before broad refactors

## docs
- `README.md` for setup
- `ARCHITECTURE.md` for design
```
