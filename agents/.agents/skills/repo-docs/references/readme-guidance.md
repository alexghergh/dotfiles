# README Guidance

## Purpose

`README.md` is the front door to the repository. It should help a new reader quickly understand what the project is, how to start it, and where to go next.

## Good content

- one short paragraph describing the project and audience
- a quick start path or first commands
- a brief top-level repository layout when structure matters
- links to deeper docs such as `ARCHITECTURE.md`, `CONTRIBUTING.md`, or deployment docs

## Avoid

- repeating the full architecture discussion
- exhaustive API contracts
- low-level code walkthroughs
- long historical narrative

## Existing-style rule

If `README.md` already has a stable tone and section structure, preserve it unless the user allows a broader rewrite.
