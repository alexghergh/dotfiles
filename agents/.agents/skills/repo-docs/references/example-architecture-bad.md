# ARCHITECTURE Bad Example

## Warning signs

- repeats setup instructions from the README
- lists every helper function
- mixes current design with speculative future redesigns
- uses broad claims that are not reflected in the code
- has no clear component boundaries or flow descriptions

## Bad pattern

```md
# architecture

The app is cleanly layered and highly scalable.

## files
- helper one
- helper two
- helper three

## future state
Later we may split this into microservices.
```
